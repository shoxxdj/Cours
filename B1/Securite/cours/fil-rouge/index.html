Fil Rouge

TP Fil Rouge

Evaluation Objectifs :
  • White Hat
    • Développer et mettre en place un site web sécurisé
  • Black Hat
    • Attaquer vos concurents
    • Auditer les sites et y trouver des failles
    • Soummetre vos bugs

Scénario

Par groupe de 5 vous representez une entreprise responsable de l'hebergement de sites web. La solution que vous proposez inclus les services suivants :

  • Mise en place d'un site web
  • Accès FTP
  • Accès SSH
  • Base de données MySQL
  • Accès à une console d'administration
  • Hebergement d'appications personnalisées
  • Extra: Développement du site

Comment ca marche ?

Le client ( moi ), vous demande le package 5 étoiles, vous devez donc mettre en place tout les services que vous proposez ( voir ci-dessus ). Le développement demandé est le suivant : Un site web dynamique, permettant aux utilisateurs de proposer la location de leur appartement pour une ou plusieures nuits. Une autre catégorie d'utilisateur doit pouvoir réserver l'appartement, ils doivent aussi pouvoir commenter les appartements dans les quels ils ont dormis. En résumé, à vous de créer le clone de Airbnb.

Evidemment tous ces services doivent etre sécurisé.

Comment avoir 20 au TP ?

Le TP se découpe en 2 parties. Dans la première vous allez concevoir le site, vous allez aussi vous occuper de la partie adminsys. Cette première partie vous rapporte 1000 points découpés comme suit :

  • 200: Démarer le serveur et le rendre accessible
  • 400: Avoir un site dynamique
  • 400: Mettre en place des outils d'administration
L'autre partie du TP concerne la sécurité, à vous de sécuriser au maximum le site et le serveur. Vous n'allez pas inventer la sécurité, les cours sont la pour ça. Cette partie vous apporte aussi 1000 points découpés comme suit :
  • +50 / semaines sans failles
  • -100 / failles
  • -500 si une autre équipe arrive à rooter votre serveur
Evidement les attaquants sont aussi récompensés :
  • + 50 / failles
  • +250 si vous rootez le serveur
- Ehhhh pourquoi ça rapporte moins d'attaquer ?
- Parce que vous etes plusieures équipes et qu'il est probable qu'une meme faille se trouve à divers endroits
A tout moment vous pouvez demander l'audit de votre plateforme. Ceci vous coutera 200 points. A vous de calculer selon votre budget.

Comment soumettre les failles trouvées ?

Pour chaque faille que vous trouvez vous devez fournir une documentation expliquant à l'équipe adverse la faille que vous avez trouvé. Ce document doit contenir des screenshots et des explications détaillés. Vous devez aussi me fournir un document expliquant la faille que vous avez trouvé et comment il est possible de la corriger. Une fois ces ellements fournis je les étudies et vous valide ( ou non ) les points.

Rendu

TP Final

Vous etes le dirigeant d'une entreprise.

Le projet va commencer et vos premiers employés vont arriver.

Vous avez un budget illimité. Cependant, de nature radine vous ne gaspillerais pas votre argent et tenterais de limiter les couts.

Votre entreprise :

  • - Vous hebergez des sites web.
  • - Vous proposez la création de site web.
  • - Vous garantissez la sécurité de l'infrastructure et des développements.
Moi :
  • Je suis un client d'une très grande entreprise et vous ne voulez pas passer à coté d'une opportunité de travailler avec moi.
  • Je souhaite faire heberger mon site de e-commerce dans votre infrastructure.
  • Je souhaite aussi que vous le développiez.
  • A vous de me convaincre.
    • Je serais attentifs aux critères suivants :
    • Sécurité de l'infrastructure
    • Sécurité des développements
    • Cout des équipements
    • Justification de ces couts.

      • ( je ne veux pas que vous me dites "c'est sécurisé, je veux savoir pourquoi et comment vous vous en assurez ")
